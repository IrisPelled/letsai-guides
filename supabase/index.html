שלב 0

פתיחת פרויקט חדש

מכינים את הסביבה ונותנים לקלוד את ההוראות הראשונות

אתם עושים — פעולה ידנית
קלוד עושה — ממתינים
הסבר / מידע
i
איך עובדים?

עובדים דרך Claude Code — אפליקציית קלוד במחשב. קלוד בונה הכל לוקאלית אצלכם. אתם לא מריצים פקודות ולא פותחים טרמינל.

1
אתם

פתחו את Explorer ונווטו לנתיב:

C: → Users → [שם המשתמש] → .claude → projects

צרו שם תיקיה חדשה עם שם הפרויקט — באנגלית, ללא רווחים

2
אתם

העתיקו לתוך התיקיה החדשה את שני קבצי התוכנית:

  • קובץ plan.md — התוכנית הראשית
  • קובץ Phase 1
3
אתם

פתחו את Claude Desktop, עדכנו את קלוד שעובדים לוקאלית ותנו לו את שם התיקיה שיצרתם

4
אתם

תנו לקלוד את plan.md ואת קובץ Phase 1 — במצב Plan Mode

בשלב זה קלוד רק קורא, מבין וכותב לעצמו תוכנית עבודה. הוא עדיין לא בונה כלום

5
אתם

קראו את התוכנית שקלוד כתב, וודאו שהיא נכונה — ולחצו Accept כדי לאשר לו לצאת לעבודה

6
קלוד

יוצר את הפרויקט ובונה את כל הקבצים — ממתינים

  • Region: השאירו ברירת מחדל

אתם

9
אתם

https://abcdefghijklmnopqr.supabase.co

שלב 1ב

הכנסת פרטי החיבור לפרויקט

מחברים את הפרויקט לבסיס הנתונים דרך קובץ .env.local

i
מה זה .env.local?

קובץ סודי שמכיל את פרטי החיבור לסופהבייס. קלוד יצר אותו ריק — עכשיו ממלאים אותו עם הערכים שהועתקו.

1
פתחו

פתחו את סייר הקבצים, נווטו לתיקיית הפרויקט, ומצאו את הקובץ .env.local בשורש התיקיה

2
פתחו

לחצו פעמיים על .env.local — בחרו Cursor או VS Code כתוכנת פתיחה

3
הדביקו

הדביקו את שלוש השורות — כל שורה בנפרד, עם Enter אחרי כל שורה:

NEXT_PUBLIC_SUPABASE_URL=https://xxxx.supabase.co NEXT_PUBLIC_SUPABASE_ANON_KEY=sb_publishable_... NEXT_PUBLIC_SUPABASE_PROJECT_ID=xxxxxxxxxxxxxxxxxxxx
שימו לב:
  • אין רווח לפני ואחרי =
  • כל שורה בשורה נפרדת — Enter אחרי כל שורה
  • אין גרשיים (") סביב הערך
4
שמרו

שמרו את הקובץ: File → Save או Ctrl + S

זו הפעם היחידה שפותחים Cursor / VS Code בכל התהליך הזה

i
בטוח

קובץ .env.local לא עולה לאינטרנט ולא נשלח לאף מקום. הפרטים נשארים רק במחשב.

שלב 1ג

חיבור ה-Supabase MCP Connector

מאפשרים לקלוד לפעול ישירות בפרויקט Supabase

i
מה זה MCP Connector?

חיבור שמאפשר לקלוד לפעול ישירות בפרויקט Supabase — ליצור טבלאות, להריץ SQL ועוד. בלעדיו קלוד לא יכול לגעת בבסיס הנתונים.

1
אתם

ב-Claude Desktop גשו ללשונית Customize ובחרו Connectors

2
אתם

מצאו את Supabase ברשימת הקונקטורים ולחצו לחבר אותו

3
אתם

הזינו את ה-Project ID שהועתק בשלב 1א וודאו שהחיבור הצליח

שלב 1ד

קלוד בונה את התשתית

קלוד מקים קבצי קוד ויוצר את הטבלאות בסופהבייס

1
אתם

תנו לקלוד את קובץ Phase 1 ובקשו ממנו לבצע אותו

הקובץ מכיל הוראות מפורטות — קלוד יקרא ויתחיל לעבוד

2
קלוד

מתקין את חבילת Supabase:

npm install @supabase/supabase-js
3
קלוד

יוצר את קבצי הקוד הבסיסיים:

lib/supabase/client.ts ← חיבור לסופהבייס lib/leads/submitLead.ts ← לוגיקת שליחת טופס supabase/schema.sql ← הגדרת טבלאות .env.example ← תבנית ריקה
4
קלוד

מריץ את ה-SQL דרך ה-MCP Connector — יוצר את הטבלאות ישירות בסופהבייס

לא נדרשת פעולה — קלוד עושה הכל אוטומטית

שלב 2

בניית ממשק המשתמש

מעלים תמונות, מספקים עיצוב, וקלוד בונה את האתר

הכנת חומרי עיצוב
i
איפה שמים את התמונות?

Next.js יוצר תיקיה בשם public/ בשורש הפרויקט. אנחנו יוצרים בתוכה assets/ לכל התמונות.

בקוד כתוב src="/assets/hero.jpg" — הנתיב מתחיל אחרי public, כי public שקופה לדפדפן.

1
אתם

פתחו את תיקיית הפרויקט, היכנסו ל-public → assets והעבירו לשם את כל התמונות והלוגו

i
אילו תמונות נדרשות?
לוגו
PNG עם רקע שקוף
logo.png
Hero / רקע ראשי
תמונה גדולה לחלק עליון
hero.jpg
פרופיל / אודות
תמונה של בעל/ת העסק
about.jpg
מוצר / שירות
תמונה לכל קטגוריה
product-1.jpg

תנו לתמונות שמות באנגלית ללא רווחים

3
אתם

קלוד בונה את האתר
